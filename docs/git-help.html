Git 常用操作指南

1. 初始化仓库

git init

作用：在当前目录初始化一个新的Git仓库

示例：

cd my-project
git init

2. 克隆远程仓库

git clone <repository-url>

作用：克隆远程仓库到本地

参数：

  • <repository-url>：远程仓库地址

示例：

git clone https://github.com/user/repo.git

3. 查看状态

git status

作用：查看当前工作目录和暂存区的状态

示例：

git status

4. 添加文件到暂存区

git add <file>

作用：将文件添加到暂存区

参数：

  • <file>：要添加的文件，可以使用.添加所有文件

示例：

git add .
git add README.md

5. 提交更改

git commit -m "commit message"

作用：将暂存区的更改提交到本地仓库

参数：

  • -m：提交信息

示例：

git commit -m "更新README文档"

6. 查看提交历史

git log

作用：查看提交历史记录

常用选项：

  • --oneline：简洁显示
  • --graph：图形化显示分支

示例：

git log --oneline --graph

7. 创建分支

git branch <branch-name>

作用：创建新分支

参数：

  • <branch-name>：分支名称

示例：

git branch feature/new-feature

8. 切换分支

git checkout <branch-name>

作用：切换到指定分支

参数：

  • <branch-name>：要切换到的分支名称

示例：

git checkout main

9. 合并分支

git merge <branch-name>

作用：将指定分支合并到当前分支

参数：

  • <branch-name>：要合并的分支名称

示例：

git checkout main
git merge feature/new-feature

10. 推送更改到远程仓库

git push origin <branch-name>

作用：将本地分支的更改推送到远程仓库

参数：

  • origin：远程仓库名称
  • <branch-name>：要推送的分支名称

示例：

git push origin main

11. 拉取远程更改

git pull origin <branch-name>

作用：从远程仓库拉取最新更改并合并到当前分支

参数：

  • origin：远程仓库名称
  • <branch-name>：要拉取的分支名称

示例：

git pull origin main

12. 回滚到指定提交

git reset --hard <commit-hash>

作用：将当前分支重置到指定提交

参数：

  • --hard：强制重置工作目录和暂存区
  • <commit-hash>：要回滚到的提交哈希值

示例：

git reset --hard a1b2c3d

13. 查看远程仓库

git remote -v

作用：查看已配置的远程仓库

示例：

git remote -v

14. 添加远程仓库

git remote add origin <repository-url>

作用：添加一个新的远程仓库

参数：

  • origin：远程仓库名称
  • <repository-url>：远程仓库地址

示例：

git remote add origin https://github.com/user/repo.git

15. 删除分支

git branch -d <branch-name>

作用：删除本地分支

参数：

  • -d：删除已合并的分支
  • -D：强制删除未合并的分支
  • <branch-name>：要删除的分支名称

示例：

git branch -d feature/old-feature

16. 查看文件差异

git diff

作用：查看工作目录和暂存区之间的差异

示例：

git diff

17. 暂存当前更改

git stash

作用：将当前工作目录的更改暂存起来

示例：

git stash

18. 应用暂存的更改

git stash apply

作用：应用最近一次暂存的更改

示例：

git stash apply

19. 创建标签

git tag <tag-name>

作用：在当前提交创建标签

参数：

  • <tag-name>：标签名称

示例：

git tag v1.0.0
